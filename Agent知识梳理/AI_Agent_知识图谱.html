第一章
AI Agent 基础概念
理解 Agent 的本质，是做好 AI 产品规划的基础前提
🤖
什么是 AI Agent？
AI Agent 是能够感知环境、自主制定计划、使用工具执行任务并持续调整行为以完成目标的 AI 系统。
📌 最简公式：Agent = LLM + 工具使用 + 记忆 + 规划 + 行动
⚡
Agent vs LLM 的本质区别
LLM 是单次推理引擎——给定输入，输出文本。Agent 是持续运行的行动系统——能调用工具、观察结果、迭代决策，直至目标达成。
📌 LLM 是 Agent 的"大脑"，但 Agent 远不止于 LLM
🔧
Agent vs Copilot
Copilot 是辅助型，需要人类主导，AI 提供建议；Agent 是自主型，可以独立完成任务闭环。二者可以演进——Copilot 逐步放开自主权后成为 Agent。
📌 产品策略：从 Copilot 起步，数据驱动演化为 Agent
🏭
Agent vs RPA / Workflow
RPA 基于规则，无法处理模糊情况。传统 Workflow 是预定义固定路径。Agent 基于语义理解，能动态生成执行路径，应对非结构化、需判断的场景。
📌 Agent = 自适应工作流 + 语义理解 + 工具执行

⚡ ReAct — 推理与行动交替
2022 年 Yao 等提出，当前最广泛使用的 Agent 推理范式。LLM 交替生成 Thought（思考）和 Action（行动），观察结果后继续循环。
# 以"查询特斯拉 Q1 交付量"为例 Thought: 用户要查询特斯拉 2025 Q1 交付量，需要搜索最新数据 Action: web_search("Tesla Q1 2025 delivery numbers") Obs: 特斯拉 2025Q1 交付 336,681 辆，同比下降约 13% Thought: 已获得准确数据，可以回答用户了 Answer: 特斯拉 2025 年 Q1 交付 336,681 辆，同比下降约 13%
🔗 Chain-of-Thought — 链式思维
通过"Let's think step by step"引导 LLM 展开推理链，减少跳跃性错误。适合数学计算、逻辑推理类任务，通常作为 ReAct 的基础组件使用。
# 关键提示技巧 Prompt: "请一步一步思考，然后给出最终答案" Step 1: 分析问题的已知条件 Step 2: 推导中间结论 Step 3: 验证逻辑一致性 Answer: 综合以上推导得出结论
🌳 Tree of Thoughts — 思维树
Google DeepMind 2023 年提出。LLM 同时探索多条推理分支并评估每条路径，选择最优解。适合需要复杂规划的任务（写作、架构设计），但计算成本显著高于 ReAct。
# 多分支并行探索 Branch A: 方案一 → 评分 7/10 Branch B: 方案二 → 评分 9/10 ← 最优 Branch C: 方案三 → 评分 5/10 Select: 选择 Branch B 继续深化执行
🪞 Reflexion — 自我反思
Shinn 等 2023 年提出。Agent 对自身输出进行批评和修正——执行任务后，由 LLM 扮演"评估者"角色生成反思，写入记忆，指导下轮改进。在 HotpotQA、AlfWorld 等基准显著优于无反思基线。
# 自我反思循环 Execute: 执行任务，得到结果（可能失败） Reflect: "我失败的原因是什么？下次应该怎么做？" Memory: 将反思结论写入长期记忆 Retry: 携带反思记忆，重新执行任务

第三章
多 Agent 系统架构
由两个以上 Agent 协作完成任务，突破单 Agent 能力边界，应对复杂场景
🔗 流水线型
Pipeline / Sequential
Agent A → Agent B → Agent C → 输出
Agent A 的输出作为 Agent B 的输入，依次传递。适合有明确顺序依赖的任务。
💡 适用：数据处理流水线、内容生产链（搜集→分析→写作→审校）
🏛️ 层级型
Hierarchical / Orchestrator-Worker
Orchestrator
↙　　↓　　↘
Worker A Worker B Worker C
Orchestrator 负责任务分解和调度，多个 Worker 各自完成子任务。最主流的生产架构。
💡 适用：软件开发（PM→架构→开发→QA）、复杂分析任务
🤝 扁平协作型
Collaborative / Peer-to-Peer
Agent A ⇌ Agent B
⇵　　　　　⇵
Agent C ⇌ Agent D
所有 Agent 地位平等，通过共享黑板或消息总线交换信息，自发协作。弹性高，调试复杂。
💡 适用：创意协作、开放性问题探索（AutoGen 框架）
🔀 混合型
Hybrid
Orchestrator
↓ (层级)
Pipeline → → → 验证 Agent
结合多种模式。整体层级型调度，局部流水线执行，验证环节扁平互查。Anthropic 推荐从简单模式起步，按需演化。
💡 适用：生产级复杂系统，通常从前三种演化而来
⭐ 核心设计模式：Orchestrator-Worker（编排者-工作者）
为什么是生产首选？
清晰的职责边界，Orchestrator 专注规划协调，Worker 专注专项执行，易于独立优化和替换。
可观测性强
Orchestrator 的决策过程可被记录和审计，满足企业合规需求，便于问题定位。
模型分级使用
Orchestrator 用强模型（Claude Opus / GPT-4o），Worker 用轻量模型（Claude Haiku / GPT-4o-mini），平衡性能与成本。
第四章
Agent 工作流关键技术
工程实践中必须掌握的核心技术点，决定 Agent 产品的可靠性
🔀
任务分解
复杂任务必须拆分为可执行原子步骤。三种方式：LLM 自主分解（动态）、硬编码 DAG（稳定）、MRKL 模块化分解。
⚠️ PM 关注：分解粒度影响 LLM 调用次数和成本，需 A/B 测试找最优粒度
📚
Agentic RAG
Agent 主动决定何时、如何检索，支持迭代式多轮检索（发现信息不足时再次检索），显著优于被动 RAG。混合检索（向量+BM25）+重排序进一步提升精度。
📌 Morgan Stanley：10 万+文档向量化，16000 顾问实时使用
👥
人机协同 HITL
高风险场景必须引入人类审核节点。策略：断点审核（关键操作前暂停）、置信度阈值（低分自动转人工）、影子模式（上线初期只建议不执行）。
⚖️ 平衡点：操作可逆性 × 影响面 → 设定合理介入阈值
🔁
错误处理与自修正
重试策略（指数退避）+ Fallback 路径（备选工具）+ Reflexion 自修正（将错误注入 Prompt 引导修正）+ 外部验证器（代码实际运行结果验证）。
🎯 最佳实践：代码类任务必须通过实际运行而非 LLM 自评来验证
🛠️
Function Calling 工程实践
工具描述质量直接影响调用准确率（最常见失效原因：描述模糊）。并行工具调用（GPT-4o / Claude 均支持）显著提升效率。写操作必须设计幂等机制。
📌 关键：工具的 name/description 越精准，LLM 调用越准确
📊
可观测性建设
生产级 Agent 必须可观测。追踪工具：LangSmith · Langfuse · Arize。评估维度：任务成功率、平均步骤数、每次成本、安全事件率、用户满意度。
⚠️ 最容易忽视但最关键——没有可观测性就没有迭代能力

Business Value Assessment

Viability 可行性
Technical Feasibility Assessment
当前 LLM 能力是否能可靠完成该任务？任务模糊性程度如何？

V
Safety 安全性
Risk & Compliance Assessment

核心术语速查
AI PM 必备词汇，可直接用于技术对话
Agent
能自主感知、规划、行动以完成目标的 AI 系统，由 LLM + 工具 + 记忆 + 规划构成
ReAct
Reasoning + Acting，LLM 交替生成思考和行动的推理范式，当前最主流的 Agent 推理方式
RAG
Retrieval Augmented Generation，检索增强生成，将外部知识实时注入 LLM 上下文
Function Calling / Tool Use
LLM 调用预定义工具/函数的机制，是 Agent 突破能力边界的核心手段
Orchestrator
多 Agent 系统中负责任务分解和调度的主 Agent，通常使用更强的模型
HITL
Human-in-the-Loop，在 Agent 关键决策点引入人类审核，平衡自主性与安全性
CoT
Chain-of-Thought，链式思维，引导 LLM 逐步展开推理过程，减少跳跃性错误
Reflexion
Agent 自我反思机制，分析失败经验并写入记忆，指导下一次改进执行
Agentic RAG
Agent 主动决定何时、如何检索的增强型 RAG，可迭代多轮检索，优于被动 RAG
Guardrails
安全护栏，对 Agent 的输入输出进行过滤和约束的安全机制（如 Bedrock Guardrails）
Prompt Injection
通过恶意输入劫持 Agent 行为的攻击方式，Agent 特有的安全威胁
SWE-bench
软件工程 Agent 能力标准测试集，衡量 Agent 解决真实 GitHub Issue 的能力

面向 AI PM 的系统性知识框架 · 内容基于公开行业报告与权威媒体报道
